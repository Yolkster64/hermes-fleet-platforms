## 🎓 LESSONS LEARNED

### Discovered During Experimentation

1. **Four is Magic** - The number 4 appears everywhere
   - 4 agents in parallel (optimal parallelism)
   - 4 core agents in 8-agent fleet (4 core + 4 support)
   - Fractal pattern across hierarchy levels

2. **Superlinear Overhead** - More isn't better
   - Coordination cost grows as O(n²)
   - Each additional agent increases coordination burden
   - 16 agents actually slower than 8 due to overhead

3. **Sweet Spots Exist** - All metrics show clear optima
   - Not on a linear curve
   - Unimodal (single peak, not plateaus)
   - Different metrics peak at same points (8 agents)

4. **Quality Diminishes** - Testing has ROI curve
   - 70%→90%: fast gains (10%/hour)
   - 90%→95%: slower (2.5%/hour)
   - 95%→97%: very slow (1%/hour)

5. **Simplicity Wins** - Elegant beats complex
   - Simpler designs have better ROI
   - Easier to understand = easier to maintain
   - Fewer moving parts = fewer failures
