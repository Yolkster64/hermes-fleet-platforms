Version Format: MAJOR.MINOR.PATCH[-PRERELEASE][+BUILD]

Versioning Rules:
• MAJOR (X): Increment for breaking API changes → 2.0.0
• MINOR (Y): Increment for new backward-compatible features → 1.1.0
• PATCH (Z): Increment for bug fixes only → 1.0.1
• PRERELEASE: Use -alpha, -beta, -rc for development → 1.0.0-beta.1
