# Version Control for AI-Generated Code

Tracking, managing, and maintaining AI-generated code in production.

## Comment Headers for AI-Generated Code

Every AI-generated code file must include a header:

```powershell
<#
.AI_GENERATED_HEADER
    Source: ChatGPT-4 / GitHub Codex
    Generated: 2024-01-15T10:30:00Z
    Model: gpt-4 / codex-v1
    Prompt Hash: abc123def456
    Generator: OpenAI Copilot
    Status: REQUIRES_REVIEW
    ReviewedBy: [Name] - [Date]
    ApprovedBy: [Name] - [Date]
#>
```

## Header Fields Explained

| Field | Purpose | Example |
|-------|---------|---------|
| Source | Which AI service | ChatGPT-4 |
| Generated | Timestamp of generation | 2024-01-15T10:30:00Z |
| Model | Specific model version | gpt-4-0613 |
| Prompt Hash | Hash of original prompt | abc123... |
| Generator | AI system used | OpenAI Copilot |
| Status | Current approval status | REQUIRES_REVIEW |
| ReviewedBy | Code reviewer info | Alice - 2024-01-15 |
| ApprovedBy | Approval authority | Bob - 2024-01-15 |

## Reverting AI Changes

### When to Revert

- Security issue discovered in AI code
- Performance degradation
- Compliance violation
- Functionality broken
- Better alternative available

### Revert Procedure

```powershell
# Step 1: Identify problematic commit
git log --oneline | grep "AI-GENERATED"

# Step 2: Create revert commit
git revert <commit-hash>

# Step 3: Document reason
# In commit message, explain WHY it was reverted

# Step 4: Notify team
# Email security team with revert reason
```

### Revert Commit Message

```
[REVERT] AI-generated consolidation script

Reason: Security vulnerability detected

- Reverted commit: abc123def
- Generated by: GitHub Codex
- Issue: Potential privilege escalation
- Severity: High
- Reported by: Security team
- Analysis: [detailed explanation]

This reverts commit abc123def456.

Co-authored-by: Human <human@company.com>
```

## Compliance Documentation

### What to Keep

- [ ] Original AI prompt
- [ ] Generated code
- [ ] Approval documentation
- [ ] Test results
- [ ] Change logs
- [ ] Audit logs
- [ ] Performance metrics
- [ ] Security reviews

### Retention Period

- AI-generated code: Keep forever
- Approval documentation: Keep 7 years (compliance)
- Audit logs: Keep 3 years (regulatory)
- Performance metrics: Keep 1 year (trending)

## Best Practices

1. **Always Mark AI Code**
   - Use standard header format
   - Include generation timestamp
   - Document original prompt

2. **Track All Changes**
   - Version every modification
   - Log who changed what and when
   - Document why changes were made

3. **Maintain Audit Trail**
   - Keep all review documents
   - Archive approval emails
   - Store performance test results

4. **Know When to Revert**
   - Security issues: Revert immediately
   - Performance problems: Revert after analysis
   - Better alternatives: Revert after validation

5. **Document Everything**
   - Explain all decisions
   - Include reasoning
   - Make audit trail clear
